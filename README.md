# COMP2322-Project1
Multi-threaded Web Server
This is a multi-threaded Web server implemented in Python using basic socket programming from scratch. It is designed to handle multiple HTTP requests concurrently and supports various HTTP methods, status codes, and header fields.
1. Project Requirements
Language: Python 3.x 
Libraries: socket, threading, os, datetime, time, calendar (Standard libraries, no external installation required)
2. File Descriptions
server.py: The core server script containing the socket initialization and request handling logic.
index.html: A sample HTML file used for testing GET requests (Text files).
test.jpg: A sample image file used for testing GET requests (Image files).
server.log: A log file generated by the server to record client request statistics.
3. How to Run the Program
Since Python is an interpreted language, no compilation is required. Follow these steps to execute the server:
Open your terminal or command prompt.
Navigate to the directory containing server.py.
Run the following command:
Bash
python3 server.py
The server will start and listen on http://127.0.0.1:8080.
4. Supported Features
HTTP Methods 
GET: Retrieves and sends requested files (HTML and images) to the client.
HEAD: Returns only the HTTP response headers without the body content.
Status Codes 
200 OK: File found and served successfully.
400 Bad Request: Malformed request or unsupported method.
403 Forbidden: Access to specific files (like server.py) is restricted.
404 Not Found: The requested file does not exist.
304 Not Modified: Cached version is still valid.
Header Fields 
Cache Control: Handles Last-Modified and If-Modified-Since headers to manage web caching.
Connection: Supports both keep-alive (persistent) and close (non-persistent) connections.
5. Logging System
Every request is recorded in server.log with the following format:
Client IP | Access Time | Requested File | Response Type
