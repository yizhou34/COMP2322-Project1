# Multi-threaded Web Server (COMP2322-Project1)

This is a multi-threaded Web server implemented in Python using basic socket programming from scratch. It is designed to handle multiple HTTP requests concurrently and supports various HTTP methods, status codes, and header fields.

## 1. Directory Structure

The repository is strictly organized as per the project requirements:

* **`src/server.py`**: The core server script containing the socket initialization, multi-threading mechanism, and HTTP request handling logic (well commented in English).
* **`test_files/`**: The directory containing static files used for testing the GET command.
  * `index.html` & `test.html`: Sample text files.
  * `test.jpg`: A sample image file to demonstrate MIME type handling.
* **`server.log`**: The request log recording client IP, access time, requested file, and response status. *(Note: This file is automatically generated in the root directory when the server runs).*
* **`README.md`**: Instructions on how to compile and run the server.

## 2. How to Run the Server

Since Python is an interpreted language, no compilation is required. Follow these steps to execute the server:

1. Open your terminal or command prompt.
2. Navigate to the **root directory** of this project (`COMP2322-Project1`).
3. Run the following command to start the server: bash python3 src/server.py
4. The server will start and listen on http://127.0.0.1:8080. It will automatically map incoming requests to the ./test_files/ directory.

### 3. Supported Features & Testing Commands

You can test the server using `curl` commands in a new terminal window.

#### HTTP Methods & Status Codes

* **200 OK (GET - Text)**: 
  `curl -i http://127.0.0.1:8080/test.html`

* **200 OK (GET - Image)**: 
  `curl -i http://127.0.0.1:8080/test.jpg`

* **200 OK (HEAD Command)**: 
  `curl -I http://127.0.0.1:8080/index.html`

* **404 Not Found** (Requesting a non-existent file): 
  `curl -i http://127.0.0.1:8080/abc.html`

* **403 Forbidden** (Security measure protecting source code): 
  `curl -i http://127.0.0.1:8080/server.py`

* **400 Bad Request** (Malformed request or unsupported method): 
  `curl -i -X WRONG http://127.0.0.1:8080/`

#### Advanced Header Processing

* **Cache Control (304 Not Modified)**:
  The server supports `Last-Modified` and `If-Modified-Since` headers. To test, replace the date string below with the actual Last-Modified time of your file:
  `curl -i -H "If-Modified-Since: Mon, 20 Apr 2026 11:31:08 GMT" http://127.0.0.1:8080/test.html`

* **Persistent Connections**: 
  The server supports `Connection: keep-alive` to process multiple requests over a single TCP connection.

4. Dependencies
Python 3.9
